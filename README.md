# 弦上一刻（String Moment）项目README
 弦上一刻（String Moment）是一个连接音乐爱好者与琴行的一站式乐器抢购与服务平台，核心聚焦热门乐器限时秒杀、普通乐器购买核心场景，提供了完整的用户、商品、订单、秒杀与地址管理模块。

项目基于SpringBoot框架独立开发，目前已完成v1.0版本核心业务模块，重点攻克秒杀场景下的库存超卖、一人一单等核心痛点，通过合理的技术选型保障业务稳定性与基础性能；

后续将逐步集成RabbitMQ、SpringCloud、AI相关能力，持续迭代升级，完善高并发处理、分布式架构与智能化服务，进一步提升系统吞吐量与用户体验，完善平台生态。

## 核心技术栈

- 后端框架：SpringBoot、MyBatis
- 数据存储：MySQL（核心业务）、Redis（缓存、秒杀库存、防重校验）
- 核心工具：Lua（原子性控制）、Spring Task（定时任务）、JWT（用户认证）



## 核心功能

- 用户模块：注册、登录（JWT认证）、个人信息查询
- 商品模块：商品浏览、详情查看、分类搜索
- 秒杀模块：活动列表/详情、资格校验、高并发下单（防超卖、一人一单）
- 订单模块：创建订单、列表/详情查询、取消订单、模拟支付、超时关单
- 地址模块：收货地址增删改查及默认地址管理



## 核心实现：秒杀功能

本项目核心解决了高并发场景下的“超卖”和“一人一单”核心痛点，通过 Redis + Lua + 定时任务 的组合方案，兼顾性能与数据一致性，具体实现如下：

### 1. 活动状态与缓存生命周期管理（SeckillStatusTask）

通过定时任务（每10秒执行）同步数据库与缓存状态，确保活动状态准确并及时释放系统资源，实现缓存生命周期的精细化管理：

- 活动开始：将商品可用库存（available_stock）加载至Redis，同时清空该活动对应的用户购买记录Set，为秒杀做好准备
- 活动结束：将Redis中剩余的可用库存同步回数据库，清理该活动相关的所有缓存键（库存键、用户购买记录Set等），避免缓存冗余

### 2. 秒杀下单流程（executeSeckill）

下单流程采用“Lua脚本原子校验 + 数据库落单”的两阶段方式，既保证高并发下的性能，又确保数据一致性，核心步骤如下：

1. 前置校验：校验活动状态、商品有效性、收货地址合法性，过滤无效请求
2. 缓存兜底：调用ensureRedisCache方法，若Redis缓存意外丢失，则从数据库恢复库存数据，提升系统鲁棒性
3. 原子校验与扣减：执行Lua脚本，原子化完成“活动时间校验 → 防重校验 → 库存扣减”三大操作，杜绝并发异常
4. 订单落库：若Lua脚本返回成功，在数据库中创建普通订单与秒杀订单记录，完成业务闭环
5. 异常回滚：若订单落库等后续步骤失败，通过try-catch机制回滚Redis中的库存与用户购买记录，避免脏数据

### 3. Lua脚本

脚本在Redis中原子性执行，是解决高并发超卖、防重问题的关键，避免多线程并发导致的逻辑异常：

输入参数： stockKey, userKey, userId, activityStatus, currentTime, startTime, endTime

核心逻辑：

1. 时间戳校验：优先于活动状态，避免状态同步延迟导致的异常
2. 校验用户是否已购买：通过sismember判断用户是否在购买记录Set中
3. 校验并扣减库存：通过decr原子操作扣减库存，库存不足则直接返回
4. 记录用户购买记录：通过sadd将用户ID加入购买记录Set，实现一人一单

返回值：0(成功), 1(库存不足), 2(重复购买), 3(未开始), 4(已结束)
